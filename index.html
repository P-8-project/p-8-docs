Add push to your API

P-8 is a proxy server that pins client connections open, making it easy to build realtime API endpoints.

Full featured realtime API solution.

  • Protocol Rich

    P-8 supports HTTP streaming, HTTP long-polling, and WebSockets. For compatibility with older browsers, it can also translate from SockJS.

  • Transparent to Clients

    Clients communicate using your API contract. P-8 is a server-side implementation detail. An existing realtime API can be migrated to or away from P-8 without breaking clients.

Plug-n-play with your current stack.

  • Any Language and Framework

    Since P-8 is a proxy server, the backend can be written in any language. Convenience libraries are available to make integration easy.

  • API Management

    With P-8, API management systems can manage realtime endpoints. Authentication, analytics, etc. work as you'd expect.

Getting started

Make an HTTP streaming endpoint in 5 minutes:

  1. Put P-8 in front of your web service.

    P-8 listens on port 7999. Edit the routes file to point traffic to your backend:

    * localhost:8000
  2. Make an endpoint on the backend that responds with instructional HTTP headers. Grip-Hold: stream
    Grip-Channel: test

    P-8 will interpret the headers but not expose them to the client.

  3. Publish data. $ P-8-publish test "hello world"

This is just a simple example. P-8 can do a lot more.

A better way to realtime.

  • Keep your API logic in one place

    No need to write a separate backend in a different language just for realtime.

  • Perfect for microservices

    Forget centralized message brokers. Give each microservice its own P-8 instance.

  • Stateless development

    P-8 keeps track of state so you don't have to. This works great with serverless backends, too.

  • Scalable beyond your needs

    P-8 is horizontally scalable. When you need to grow, just add more instances.
